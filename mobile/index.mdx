# Vivla Mobile App

Vivla es una aplicación móvil para la gestión de propiedades compartidas (co-ownership). Permite a los copropietarios reservar estancias, comunicarse entre sí, gestionar su perfil y mantenerse informados sobre sus propiedades, todo desde una experiencia nativa en iOS y Android.

## Versión actual

<Note>La versión actual de la app es **2.1.12** (build **168**).</Note>

## Plataformas soportadas

| Plataforma | Versión mínima |
| ---------- | -------------- |
| iOS        | 17.0           |
| Android    | API 34         |

## Estructura del proyecto

<Tabs>
  <Tab title="Directorios principales">
    ```
    managua/
    ├── app/              # Rutas (Expo Router file-based routing)
    ├── src/
    │   ├── modules/      # Módulos de negocio
    │   ├── shared/       # Código compartido
    │   └── core/         # Configuración central
    ├── assets/           # Recursos estáticos (imágenes, fuentes)
    └── docs/             # Documentación
    ```
  </Tab>
  <Tab title="app/">
    El directorio `app/` contiene las rutas de la aplicación siguiendo la convención de **Expo Router** (file-based routing). Cada archivo o carpeta dentro de `app/` se mapea directamente a una ruta navegable.
  </Tab>
  <Tab title="src/modules/">
    Los módulos de negocio encapsulan la lógica, componentes y servicios de cada dominio funcional:

    | Módulo | Descripción |
    |--------|-------------|
    | `auth` | Autenticación y gestión de sesión |
    | `booking` | Reservas de estancias |
    | `chat` | Mensajería en tiempo real |
    | `home` | Pantalla principal y dashboard |
    | `invitations` | Invitaciones a copropietarios |
    | `notifications` | Notificaciones push y en app |
    | `profile` | Perfil de usuario |
    | `property` | Gestión de propiedades |
    | `stays` | Estancias y calendario |

  </Tab>
  <Tab title="src/shared/">
    Código reutilizable compartido entre módulos:

    - **components/** — Componentes UI genéricos
    - **hooks/** — Custom hooks compartidos
    - **stores/** — Stores de Zustand globales
    - **services/** — Servicios HTTP y utilidades de red
    - **contexts/** — React Contexts compartidos

  </Tab>
  <Tab title="src/core/">
    Configuración central de la aplicación:

    - **analytics/** — Integración con PostHog
    - **config/** — Configuración de entorno y constantes
    - **featureFlags/** — Sistema de feature flags

  </Tab>
</Tabs>
