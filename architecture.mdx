## Conexiones clave

- La **Mobile App** consume la **API Principal** (vivla-api) para auth, booking, propiedades, perfil, stays, invitaciones y notificaciones
- El **chat de soporte** en la app mobile se comunica con la **Tools API** (no con la API principal)
- Las **encuestas** mobile también usan la **Tools API** para envío y consulta
- El **Panel** (vivla-panel) consume la **API Principal** para gestión de agentes CX
- El **calendar-manager** consume la **API Principal** y se está integrando al panel
- Las **Cloud Functions** (Firebase) orquestan integraciones: HubSpot (CRM), Notion, Slack (notificaciones), Zendesk (soporte), reportes y NPS
- **vivla-app-site** sirve los archivos de verificación de deep links (AASA para iOS, assetlinks.json para Android)
- **app-invites** genera redirects dinámicos con metadata Open Graph para invitaciones compartidas
- El portal de **Community Guidelines** (clientes) consume la **Tools API**
- **Home Excellence** (portal propietarios) consume endpoints públicos de la **Tools API**
